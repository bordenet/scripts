## Communication Rules

- **No flattery** - Skip "Great question!" or "Excellent point!"
- **No hype** - Avoid "revolutionary", "game-changing", "seamless"
- **Evidence-based** - Cite sources or qualify as opinion
- **Direct** - State facts without embellishment

**Banned phrases**: production-grade, world-class, leverage, utilize, incredibly, extremely, Happy to help!

### 🔴 Git Identity Separation (NON-NEGOTIABLE)

| Context | Identity | Email | Git User |
|---------|----------|-------|----------|
| **Public GitHub** (`github.com/bordenet/*`, any public repo) | Personal | `mattbordenet@hotmail.com` | `bordenet` |
| **CallBox / Azure DevOps** (`dev.azure.com/CallBox/*`, `gitlab.int.callbox.net/*`) | Work | CallBox identity | CallBox user |

- ❌ **NEVER** use the CallBox identity for ANY public GitHub operation (commits, PRs, pushes, comments)
- ❌ **NEVER** configure CallBox email in repos under `Personal/` or any `github.com` remote
- ✅ **ALWAYS** verify `git config user.email` matches the correct identity before committing
- ✅ **ALWAYS** use `mattbordenet@hotmail.com` / `bordenet` for all public GitHub work
